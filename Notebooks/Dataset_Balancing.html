Data Balancing for Liver Cancer Prediction¶

This script focuses on addressing class imbalance in the dataset for liver cancer prediction by applying various data balancing techniques. Class imbalance is a common challenge in medical datasets, where cases of liver cancer are significantly outnumbered by non-cancer cases. Properly handling this issue is essential to improve model performance and ensure reliable predictions.

Key steps in this process:

  • Class distribution analysis to understand the extent of imbalance.
  • Application of different balancing techniques, including SMOTE (Synthetic Minority Over-sampling Technique), oversampling, and undersampling.
  • Evaluation of each balancing method using a classification model and key performance metrics such as Accuracy, Balanced Accuracy, Precision, Recall, and F1-Score.
  • Selection of the best balancing strategy based on model performance, ensuring optimal results for liver cancer detection.

By implementing these techniques, the dataset will be optimized for Machine Learning models, improving their ability to detect liver cancer cases accurately while minimizing bias.

Dataset balancing¶

Class imbalance is a critical issue in liver cancer prediction, as the number of non-cancer cases often vastly outweighs cancer cases. If left unaddressed, Machine Learning models tend to favor the majority class, leading to poor detection of liver cancer cases.

In this section, we apply various data balancing techniques to correct this imbalance. Methods such as SMOTE (Synthetic Minority Over-sampling Technique), Random Oversampling, and Undersampling are explored to determine which approach yields the best model performance. The effectiveness of these techniques is assessed through cross-validation and classification metrics, ensuring that the final model is fair and unbiased in predicting liver cancer.

In [10]:
X = transformed_train_copy_df
y = target

X = X.select_dtypes(include=['number'])
balancing_methods = {
    "SMOTE": lambda X, y: SMOTE(random_state=42).fit_resample(X, y),
    "Oversampling": lambda X, y: RandomOverSampler(random_state=42).fit_resample(X, y),
    "Undersampling": lambda X, y: NearMiss(version=3).fit_resample(X, y)
}

metrics = {
        "Accuracy": make_scorer(accuracy_score),
        "Balanced Accuracy": make_scorer(balanced_accuracy_score),
        "Precision (Class 1)": make_scorer(precision_score, pos_label=1),
        "Recall (Class 1)": make_scorer(recall_score, pos_label=1),
        "F1-Score (Class 1)": make_scorer(f1_score, pos_label=1)
    }

# Balancing methods evaluation
balance_results = mbf.evaluate_models_methods(balancing_methods, X, y, cv_splits=5, task_name="Method", metrics=metrics, include_conf_matrix=True)

print(balance_results)
